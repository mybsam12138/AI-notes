# AI Workflow Summary

## Definition

**AI Workflow** = a structured sequence of steps where an AI system processes input, makes decisions, and produces output, often involving models, data, and external tools.

---

## Core Components

1. **Input**
    - User query or data

2. **Processing**
    - LLM reasoning
    - Embedding generation
    - Data transformation

3. **Decision**
    - Whether to answer directly or take further action
    - May involve conditional logic or model-based reasoning

4. **Execution**
    - Tool/API calls
    - Database queries
    - External system interaction

5. **Output**
    - Final response returned to the user

---

## Key Idea

A workflow defines **how different steps are connected and executed** to transform input into output.

---

## Types of AI Workflows

### 1. RAG Workflow (Retrieval-Augmented Generation)
- Query → Embedding → Retrieve → Inject Context → Generate Answer

### 2. Agent Workflow (Dynamic)
- Think → Decide → Act (Tool) → Observe → Repeat

### 3. AI-Assisted Development Workflow
- Prompt → Generate Code → Review → Refine → Integrate

---

## Workflow vs Agent

- **Workflow** = predefined sequence of steps (static)
- **Agent** = system that dynamically decides and executes workflows

---

## Characteristics

- Step-by-step execution
- May be static or dynamic
- Can integrate multiple components (LLM, DB, APIs)
- Focus on process orchestration

---

## One-Line Summary

AI workflow is the step-by-step process that connects models, data, and tools to transform input into a final output.
